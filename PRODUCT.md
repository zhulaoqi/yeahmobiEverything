# Yeahmobi Everything — 产品介绍

> 面向全员的 AI 技能平台，让每个岗位都有专属 AI 助手

---

## 这是什么？

Yeahmobi Everything 是一款企业级 AI 桌面应用。它不是一个通用聊天机器人，而是一个**以 Skill（技能）为核心的工作效率平台**。

每个 Skill 就是一个为特定工作场景量身定制的 AI 助手。你只需选择合适的 Skill，用自然语言描述需求，AI 就能基于预设的专业 Prompt 和公司知识库，输出高质量的工作成果。

```
你 → 选择 Skill → 自然语言描述需求 → AI 输出专业结果
```

支持 Windows 和 macOS 双平台，安装即用。

---

## 能帮你做什么？

### 开箱即用的默认 Skill

应用预置了 4 个高频 Skill，覆盖大多数员工的日常工作场景：

---

#### 翻译助手

**适用岗位**：商务、运营、产品、市场、技术

**它能做什么**：
- 中英文双向翻译，保留原文语义和专业术语
- 广告素材、商务邮件、合同条款的本地化翻译
- 批量翻译产品描述、FAQ、用户评价

**使用示例**：

> **你**：把以下广告文案翻译成英文，保持营销语气：「限时特惠，全场低至3折，新用户首单再减20元」  
> **AI**：Limited-Time Offer: Up to 70% off storewide. New users enjoy an extra $3 off on their first order.

> **你**：翻译这封商务邮件为日文，语气正式：「感谢贵司的合作意向，附件为我方的合作方案……」  
> **AI**：（日文正式商务邮件输出）

---

#### 文案撰写

**适用岗位**：市场、运营、品牌、内容

**它能做什么**：
- 撰写广告投放文案（Facebook、Google、TikTok 各平台风格）
- 生成产品描述、落地页文案、社交媒体帖文
- 润色和改写现有文案，调整语气和风格

**使用示例**：

> **你**：为一款跨境电商 App 写 3 条 Facebook 广告文案，突出免运费和限时折扣  
> **AI**：  
> 1. Free Shipping Worldwide! Shop now and save up to 50% — limited time only...  
> 2. Why pay for shipping? Get your favorites delivered FREE...  
> 3. Flash Sale Alert! ...

> **你**：帮我把这段产品描述改得更吸引人：「这是一个数据分析工具，可以帮你看报表」  
> **AI**：一键洞察业务全貌——实时报表、智能归因、趋势预测，让数据驱动每一个决策。

---

#### 代码助手

**适用岗位**：研发、测试、数据工程

**它能做什么**：
- 根据需求描述生成代码片段（Java、Python、SQL、JS 等）
- 解释和优化现有代码
- 编写单元测试、SQL 查询、正则表达式
- 排查错误信息，给出修复建议

**使用示例**：

> **你**：写一个 Java 方法，将 List\<Map\<String, Object\>\> 按照 "createdAt" 字段倒序排列  
> **AI**：
> ```java
> list.sort((a, b) -> {
>     Comparable valA = (Comparable) a.get("createdAt");
>     Comparable valB = (Comparable) b.get("createdAt");
>     return valB.compareTo(valA);
> });
> ```

> **你**：这段 SQL 查询很慢，帮我优化：SELECT * FROM orders WHERE status = 'pending' AND created_at > '2025-01-01'  
> **AI**：建议添加联合索引 `(status, created_at)` 并避免 `SELECT *`...

---

#### 数据分析

**适用岗位**：数据分析、运营、产品、商业化

**它能做什么**：
- 根据业务场景生成 SQL 查询语句
- 解读数据指标，给出分析思路
- 生成数据报告框架和可视化建议
- 帮助设计 A/B 测试方案和归因模型

**使用示例**：

> **你**：帮我写一个 SQL 查询：统计过去 7 天每个渠道的新增用户数和次日留存率  
> **AI**：
> ```sql
> WITH new_users AS (...),
>      retention AS (...)
> SELECT channel, COUNT(*) AS new_users, ...
> ```

> **你**：上周 DAU 下降了 15%，可能的原因有哪些？应该从哪些维度排查？  
> **AI**：建议从以下 5 个维度排查：1. 渠道来源变化 2. 版本更新影响 3. 推送策略调整...

### 知识库增强型 Skill（管理员创建）

除了默认 Skill，管理员可以为公司特定场景创建**知识型 Skill**。这类 Skill 绑定了公司内部文档（PDF、Markdown、TXT），AI 回答时会自动检索知识库内容作为参考，确保回答准确且符合公司实际。

以下是一些典型的知识型 Skill 示例：

---

#### 公司政策问答

**绑定知识库**：员工手册.pdf、考勤制度.md、报销规则.txt

> **你**：出差住宿标准是多少？  
> **AI**：根据《差旅费报销管理规定》第三章，国内出差住宿标准如下：一线城市不超过 500 元/晚，二线城市不超过 350 元/晚...

> **你**：年假怎么计算？  
> **AI**：根据《员工手册》第 5.2 节，入职满 1 年享 5 天年假，满 3 年享 8 天...

---

#### 产品 FAQ 客服

**绑定知识库**：产品FAQ.md、常见问题汇总.txt、退换货政策.pdf

> **你**：用户问：订单发了三天还没到怎么办？  
> **AI**：您好，国内标准快递一般 3-5 个工作日送达。您可以在「我的订单」中查看物流信息。如超过 5 个工作日未收到，请联系在线客服申请查件...

---

#### SDK 接入指南

**绑定知识库**：SDK接入文档.md、API参考手册.pdf、CHANGELOG.txt

---

#### 新人培训助手

**绑定知识库**：入职指南.pdf、部门职责.md、系统账号申请流程.txt

> **你**：我刚入职，怎么申请 VPN 账号？

## 管理员能做什么？

如果你是管理员，可以通过管理端持续扩展平台能力：

### 创建新 Skill

通过可视化向导，无需写代码即可创建 Skill：

1. **选择类型** — 通用型（纯 Prompt 驱动）或知识型（Prompt + 知识库）
2. **配置信息** — 填写名称、描述、分类、图标
3. **设置 Prompt 模板** — 定义 AI 的行为和回答风格
4. **绑定知识库**（知识型）— 上传 PDF / Markdown / TXT 文件，或手动输入知识内容
5. **发布** — Skill 即刻上线，全员可用

### 管理知识库

- 上传、更新、删除知识库文件
- 将知识文件绑定到一个或多个 Skill
- 支持 PDF、Markdown、TXT 格式
- 更新知识库后，用户下次对话自动使用最新内容

### 处理用户反馈

- 查看全员提交的反馈和 Skill 需求
- 反馈自动推送至飞书群，管理员实时感知
- 标记反馈处理状态，形成闭环

## Skill 的两种类型

| 类型 | 驱动方式 | 适用场景 | 示例 |
|------|---------|---------|------|
| **通用型 Skill** | 纯 Prompt 模板驱动 | 通用工作场景，无需特定知识 | 翻译助手、代码助手、文案撰写 |
| **知识型 Skill** | Prompt + 知识库 RAG | 需要基于公司内部文档回答 | 公司政策问答、SDK 指南、产品 FAQ |

**通用型 Skill** — AI 根据 Prompt 模板理解你的意图并生成回答，适合翻译、写作、编程等通用能力。

**知识型 Skill** — AI 在回答前会先检索绑定的知识库文档，将相关内容作为参考上下文。确保回答基于公司真实资料，而非 AI 的通用知识，适合制度查询、技术文档、FAQ 等场景。

## 数据与隐私

| 项目 | 说明 |
|------|------|
| 对话历史 | 存储在你的电脑本地（SQLite），不上传到外部 |
| 收藏与使用记录 | 存储在本地 |
| 用户偏好设置 | 存储在本地 |
| 登录会话 | 本地 + 服务端 Redis 双写，支持离线自动登录 |
| 知识库文件 | 存储在公司内网数据库，不外传 |

应用采用胖客户端架构，通过公司内网直连数据库和缓存服务，不经过公网，数据全程在企业内网环境中流转。

## 常见问题

**Q：我能用它替代 ChatGPT 吗？**  
A：它的定位不是通用聊天，而是**岗位定制化的 AI 工具**。每个 Skill 都有专业的 Prompt 模板，输出质量针对特定场景优化，比直接用通用 AI 更高效、更准确。

**Q：AI 的回答是否可靠？**  
A：对于知识型 Skill，AI 回答时会引用绑定的知识库内容，确保基于公司真实资料。通用型 Skill 基于大模型通用能力，建议对关键内容进行人工确认。

**Q：找不到我需要的 Skill 怎么办？**  
A：点击应用中的「反馈」按钮提交你的需求，管理员会收到飞书通知并评估是否创建新 Skill。

**Q：对话记录安全吗？**  
A：对话历史保存在你电脑本地的数据库中，不会上传到外部服务器。

**Q：没有网络能用吗？**  
A：AI 对话需要网络连接（调用大模型 API）。但历史记录查看、收藏管理等本地功能可离线使用，登录状态也会在本地保持。

**Q：管理员如何创建知识型 Skill？**  
A：进入管理端 → 创建 Skill → 选择「知识型」 → 上传知识文档或手动输入内容 → 配置 Prompt → 发布即可。
